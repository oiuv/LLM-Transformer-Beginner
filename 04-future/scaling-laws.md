# Scaling Laws：模型大小、数据量与计算量的关系

> 本章目标：理解 LLM 训练中"模型该多大、数据该多少、计算预算怎么分配"的核心规律。学完本章后，你应能根据硬件条件做出合理的训练决策。

---

## 1. 为什么需要 Scaling Laws？

### 1.1 核心问题

```
你想训练一个 LLM，手头有 8 张 A100 GPU，预算 10 万美元。

问题：
├── 训多大的模型？1B？7B？70B？
├── 用多少数据？10B tokens？100B tokens？1T tokens？
├── 训多久？1 天？1 周？1 个月？
└── 怎么分配这些资源效果最好？

没有 Scaling Laws，你只能瞎猜。
有了 Scaling Laws，你能算出最优配置。
```

### 1.2 直觉理解

```
模型参数 = 大脑容量
训练数据 = 教材
计算量   = 学习时间

三者的关系：
├── 大脑太小，教材再多也记不住（参数不足）
├── 教材太少，大脑再大也学不到东西（数据不足）
├── 学习时间太短，教材没读完就停了（训练不充分）
└── 学习时间太长，教材翻来覆去看也没有新收获（过拟合）

Scaling Laws 告诉你：这三者的最优配比是什么。
```

## 2. OpenAI Scaling Laws（2020）

### 2.1 核心发现

OpenAI 在 2020 年的论文《Scaling Laws for Neural Language Models》中发现：

```
模型的 loss（损失）可以用一个幂律公式描述：

L(N) = (N_c / N)^α_N

其中：
- N = 模型参数量
- N_c = 常数
- α_N ≈ 0.076

含义：参数量翻倍，loss 固定下降一定比例。
      这个关系非常稳定，从 10M 到 10B 都成立。
```

用大白话说：

```
模型越大 → loss 越低 → 效果越好
而且这个"更好"是可以精确预测的。
```

### 2.2 三个关键结论

```
结论一：模型大小和数据量都重要
├── 只加大模型不加数据 → 效果提升有限
├── 只加数据不加大模型 → 效果提升有限
└── 两者同时增大 → 效果持续提升

结论二：模型大小比数据量更重要（OpenAI 2020 的观点）
├── 同样的计算预算，优先加大模型
├── 数据可以稍微少一点
└── （后来被 Chinchilla 推翻了）

结论三：存在最优的计算分配
├── 给定计算预算，存在唯一的最优模型大小和数据量
├── 偏离这个最优配比，同样的钱效果更差
└── Scaling Laws 能算出这个最优配比
```

## 3. Chinchilla Scaling Laws（2022）— 更重要的发现

### 3.1 核心结论

DeepMind 在 2022 年的论文《Training Compute-Optimal Large Language Models》中训练了 Chinchilla（70B）模型，发现：

```
最优配比：模型参数量和训练数据量应该等比例增长

具体来说：
├── 每 1B 参数 → 需要约 20B tokens 的训练数据
├── 7B 模型 → 需要约 140B tokens
├── 70B 模型 → 需要约 1.4T tokens
└── 这个比例在很大范围内都成立
```

### 3.2 推翻了 OpenAI 2020 的结论

```
OpenAI 2020 说：优先加大模型
Chinchilla 2022 说：模型和数据要等比例增长

实际情况：
├── GPT-3（175B）只用了 300B tokens 训练
├── Chinchilla（70B）用了 1.4T tokens 训练
├── Chinchilla 参数量只有 GPT-3 的 40%
└── 但 Chinchilla 效果比 GPT-3 好！

原因：GPT-3 是"大模型少吃"，Chinchilla 是"中等模型多吃"
     Chinchilla 证明了"多吃"比"大脑袋"更重要。
```

### 3.3 实际影响

```
Chinchilla 之后，所有大模型都调整了训练策略：

GPT-3 (2020):    175B 参数, 300B tokens   → 训练不足
Chinchilla (2022): 70B 参数, 1.4T tokens  → 计算最优
Llama-1 (2023):    7B 参数, 1T tokens     → 接近最优
Llama-2 (2023):    7B 参数, 2T tokens     → 略微过度训练
Qwen-3 (2025):     8B 参数, 36T tokens    → 大幅过度训练

为什么后来又"过度训练"了？
├── 推理成本：小模型推理更便宜，训练多花点钱但部署省钱
├── 数据充足：互联网上有足够多的高质量数据
└── 边际收益：过度训练到 4-5 倍仍有明显收益
```

## 4. Scaling Laws 的数学公式

### 4.1 基本公式

```
L(N, D) = E + A/N^α + B/D^β

其中：
- L = 模型的 loss（越低越好）
- N = 模型参数量
- D = 训练数据的 token 数
- E = 不可约 loss（数据本身的熵，无法消除）
- A, B, α, β = 通过实验拟合的常数

含义：
- A/N^α：模型太小带来的误差（参数不足）
- B/D^β：数据太少带来的误差（数据不足）
- E：数据本身的噪声，再大的模型也消除不了
```

### 4.2 读懂数字

```
典型数值（来自 Chinchilla 论文）：
├── E ≈ 1.69（数据本身的最低 loss）
├── α ≈ 0.34（参数量的影响系数）
├── β ≈ 0.28（数据量的影响系数）

代入计算：
├── 1B 模型 + 20B tokens → loss ≈ 2.40
├── 7B 模型 + 140B tokens → loss ≈ 2.05
├── 70B 模型 + 1.4T tokens → loss ≈ 1.85
└── 模型越大、数据越多，loss 越接近 E（1.69）
```

### 4.3 计算最优配比

```
给定计算预算 C（FLOPs），最优的模型大小和数据量：

N_opt ∝ C^0.5    （模型参数量 ∝ 计算预算的平方根）
D_opt ∝ C^0.5    （数据量 ∝ 计算预算的平方根）

等价表述：
D_opt ≈ 20 × N_opt   （数据 token 数 ≈ 20 × 参数量）

示例：
├── 预算 10^21 FLOPs → 最优 1B 模型 + 20B tokens
├── 预算 10^23 FLOPs → 最优 10B 模型 + 200B tokens
└── 预算 10^24 FLOPs → 最优 70B 模型 + 1.4T tokens
```

## 5. 实际应用：怎么用 Scaling Laws 做决策

### 5.1 决策流程

```
你有一张 4090 Laptop（16GB 显存），想训练一个 LLM。

Step 1: 确定能训多大的模型
├── 16GB 显存 → 最大约 1B 参数（bf16）
├── 用 QLoRA → 可以训 7B（但只是微调）
└── 从零预训练 → 最多 1B

Step 2: 用 Scaling Laws 算最优数据量
├── 1B 模型 → 最优数据量 ≈ 20B tokens（约 10GB 文本）
├── 但你的计算预算有限，可能训不完 20B tokens
└── 实际可用数据量 = 计算预算 / 每 token 训练成本

Step 3: 做取舍
├── 训练不充分（数据用不完）→ 模型能力没发挥出来
├── 过度训练（数据用太多）→ 浪费计算，但效果仍会提升
└── 通常宁可"过度训练"也不要"训练不足"
```

### 5.2 minimind 的选择

```
minimind-3：64M 参数，训练数据约 1.2B tokens

Chinchilla 最优配比：64M × 20 = 1.28B tokens
minimind 实际用的：约 1.2B tokens

结论：minimind 的数据量接近 Chinchilla 最优！
     这不是巧合，作者有意控制的。
```

### 5.3 你的 chinese-gpt

```
chinese-gpt 默认配置：124M 参数

Chinchilla 最优：124M × 20 = 2.48B tokens
mini 数据集：约 200M tokens

结论：数据量只有最优的 8%，属于"严重训练不足"
     这就是为什么效果一般——不是模型太小，是数据太少。

改进方案：
├── 用完整版数据集（约 2B tokens）→ 接近最优
├── 或者缩小模型（用 -L 6 -E 512）→ 匹配现有数据量
└── 或者过度训练（多跑几个 epoch）→ 效果仍会提升
```

## 6. Scaling Laws 的局限

```
Scaling Laws 不是万能的：

1. 只预测 loss，不预测具体能力
   ├── loss 低 ≠ 会做数学题
   ├── loss 低 ≠ 会写代码
   └── loss 只是"下一个词预测的准确度"

2. 不考虑数据质量
   ├── Scaling Laws 假设数据质量恒定
   ├── 实际上 1GB 高质量数据 > 10GB 低质量数据
   └── 数据清洗和筛选不在 Scaling Laws 的范围内

3. 不考虑架构差异
   ├── Scaling Laws 对 Transformer 最准
   ├── MoE、线性注意力等架构的 Scaling Laws 不同
   └── 不同架构之间不能直接比较

4. 超大模型可能偏离
   ├── Scaling Laws 在 10M-10B 范围内最准
   ├── 超过 100B 可能出现"涌现能力"（Scaling Laws 预测不到的）
   └── 例如：GPT-4 在某个规模突然学会了做数学题
```

---

## 7. 涌现能力（Emergent Abilities）

### 7.1 什么是涌现

```
涌现：模型规模超过某个阈值后，突然学会之前不会的能力。

例如：
├── 1B 模型：不会做三位数加法
├── 7B 模型：不会做三位数加法
├── 13B 模型：突然会做了！（涌现）
└── Scaling Laws 无法预测这个"突然"的时刻

涌现的能力包括：
├── 多步推理（chain-of-thought）
├── 数学计算
├── 代码生成
├── 指令遵循
└── 多语言翻译
```

### 7.2 对你的意义

```
你的 64M 模型不会做数学题、不会写代码 → 不是训练不够，是太小了
这些能力需要模型达到一定规模才会"涌现"

这就是为什么：
├── minimind（64M）能对话但不能推理 → 规模不够
├── Qwen3-7B 能推理、能写代码 → 规模够了，涌现了
└── GPT-4 几乎什么都会 → 规模更大，涌现更多能力
```

## 小结

| 要点 | 内容 |
|------|------|
| **Scaling Laws** | 模型 loss 可以用参数量和数据量的幂律公式精确预测 |
| **Chinchilla 最优配比** | 数据 token 数 ≈ 20 × 参数量 |
| **OpenAI vs Chinchilla** | OpenAI 说优先加大模型，Chinchilla 说模型和数据等比例增长 |
| **过度训练** | 数据超过最优量仍有收益，推理时小模型更省钱 |
| **实际应用** | 先确定硬件能训多大的模型，再用 Scaling Laws 算最优数据量 |
| **局限** | 只预测 loss，不预测具体能力；不考虑数据质量和涌现能力 |
| **涌现能力** | 模型够大时突然学会的能力，Scaling Laws 无法预测 |
