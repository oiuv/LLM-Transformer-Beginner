# LLM 训练全流程：每一步在教模型什么？

> 本章目标：用 minimind 的完整训练链路，串讲 LLM 从零到可用的每一步在教模型什么能力。学完本章后，你应能清晰回答"为什么需要这个阶段"。

---

## 1. 全景图：模型的"成长路线"

```
一个 LLM 的成长过程，类比一个人的成长：

阶段          类比                    教模型什么
──────────────────────────────────────────────────
Pretrain      上学读书                知识（语言、事实、逻辑）
SFT           社会化训练              行为（怎么对话、什么格式）
LoRA          专业培训班              特定领域知识 + 行为
DPO/PPO       价值观引导              偏好（什么是好回答）
Agent RL      实习工作                技能（使用工具）
蒸馏          名师带徒                从大模型转移知识
──────────────────────────────────────────────────
```

## 2. Pretrain：教知识

### 2.1 在教什么

```
Pretrain 是模型学习"世界知识"的唯一主要阶段。

它学到的知识包括：
├── 语言知识：语法规则、句式结构、词义搭配
├── 世界知识：事实、概念、人物、事件
├── 逻辑知识：因果关系、推理模式
├── 代码知识：编程语言语法、API 用法
└── 常识：物理规律、社会常识

这些知识全部来自训练数据中的文本。
```

### 2.2 怎么教的

```
训练方式：下一个词预测（Next Token Prediction）

输入："Python 是一种"
目标："编程"

模型做的事：
├── 看到 "Python 是一种"
├── 预测下一个词："编程"
├── 和真实答案对比，算差距（loss）
├── 调整参数，下次预测更准
└── 重复数十亿次

这个过程中，模型被迫"理解"文本才能准确预测：
├── 要预测"25°C"，必须知道"北京今天晴"后面通常接温度
├── 要预测"Guido"，必须知道 Python 的创造者是谁
└── 要预测"return"，必须知道函数定义后面是什么
```

### 2.3 minimind 的 Pretrain

```
数据：pretrain_t2t_mini.jsonl（1.2GB 纯文本）
数据格式：{"text": "今天天气不错，阳光明媚..."}
训练步数：约 39,695 步 × 2 epochs
loss 变化：7.52 → 1.64

训练完后模型能做：
✅ 续写文本（"今天天气" → "今天天气不错，适合出门散步..."）
✅ 基本语法（句子通顺、标点正确）
❌ 对话（不知道怎么回答问题）
❌ 遵循指令（不知道"用户在问我"）
```

### 2.4 知识量和模型大小的关系

```
64M 模型 + 1.2B tokens：
├── 能学会基本语言能力
├── 能记住少量常见知识
└── 记不住专业领域知识

7B 模型 + 1.4T tokens：
├── 能学会复杂的语言能力
├── 能记住大量世界知识
└── 能做基本推理

知识存储在哪里？
├── FFN 层（前馈网络）是主要的"知识存储"位置
├── Attention 层主要做"知识检索和关联"
└── 参数越多，能存储的知识越多
```

## 3. SFT：教行为

### 3.1 在教什么

```
SFT 不是在教新知识，是在教模型"怎么使用已有的知识"。

它教的行为包括：
├── 对话格式：用户问什么，应该怎么回答
├── 回答风格：简洁还是详细、正式还是口语
├── 指令遵循："用表格回答" → 输出表格
├── 拒绝不当请求："帮我写病毒" → 拒绝
└── 不知道就说不知道（而不是编造）
```

### 3.2 和 Pretrain 的本质区别

```
Pretrain 的数据：
{"text": "Python 是 Guido van Rossum 创造的编程语言"}
→ 模型学到：这些词的顺序关系
→ 模型不知道：这是一个"问答"场景

SFT 的数据：
{"conversations": [
  {"role": "user", "content": "Python 是谁创造的？"},
  {"role": "assistant", "content": "Python 由 Guido van Rossum 创造"}
]}
→ 模型学到：看到 user 提问 → assistant 应该回答
→ 模型知道：这是一个"对话"场景

关键区别：SFT 教会了模型"角色意识"
├── user 是提问的
├── assistant 是回答的
├── system 是设定规则的
└── 模型学会了在 assistant 的位置给出回答
```

### 3.3 SFT 也在悄悄学知识

```
虽然 SFT 的主要目的是教行为，但 SFT 数据里的回答也包含知识：

{"role": "assistant", "content": "Python 由 Guido van Rossum 于 1991 年创造"}

模型在学"怎么回答"的同时，也记住了"1991 年"这个事实。

但 SFT 的知识学习效率远低于 Pretrain：
├── Pretrain 数据量：1.2GB（纯文本，信息密度高）
├── SFT 数据量：1.7GB（对话格式，标签占空间）
├── SFT 的有效信息密度只有 Pretrain 的 1/3 左右
└── 所以"学知识靠 Pretrain，学行为靠 SFT"
```

### 3.4 minimind 的 SFT

```
数据：sft_t2t_mini.jsonl（1.7GB 对话数据）
训练步数：比 Pretrain 少（数据量小）
学习率：1e-5（比 Pretrain 的 5e-4 小 50 倍）

为什么学习率要小？
├── Pretrain 从零学，需要大学习率快速收敛
├── SFT 在 Pretrain 基础上微调，学习率太大的会"冲掉"已有知识
└── 类比：Pretrain 是从零学画画，SFT 是调整画风，力度要轻

训练完后模型能做：
✅ 回答问题（"你是谁" → "我是..."）
✅ 遵循基本指令（"翻译成英文" → 输出英文）
✅ 结构化输出（列表、分段）
❌ 复杂推理（数学题做不对）
❌ 使用工具（不知道怎么调 API）
```

## 4. LoRA：教特定知识

### 4.1 在教什么

```
LoRA 教的是"特定领域的知识和行为"。

例如：
├── lora_identity.jsonl → 教模型"我是雪风"（特定身份）
├── lora_medical.jsonl → 教模型医学知识（特定领域）
├── lora_exam.jsonl → 教模型考试答题技巧（特定技能）
└── lora_zig.jsonl → 教模型 Zig 语言语法（特定知识）
```

### 4.2 和 SFT 的区别

```
SFT：用大量通用数据，全面调整模型行为
     几万到几十万条数据，训练小时级
     风险：可能"忘记"Pretrain 学的知识

LoRA：用少量特定数据，只训练 1-2% 的参数
     50-200 条数据，训练分钟级
     优势：不影响原有知识，只"叠加"新能力
```

### 4.3 知识 vs 行为

```
LoRA 可以同时教知识和行为：

教知识的 LoRA：
{"conversations": [
  {"role": "user", "content": "Zig 语言怎么定义变量？"},
  {"role": "assistant", "content": "用 const 或 var：const x: i32 = 42;"}
]}
→ 模型记住了 Zig 的变量定义语法（知识）

教行为的 LoRA：
{"conversations": [
  {"role": "user", "content": "你好"},
  {"role": "assistant", "content": "你好，我是雪风，有什么可以帮你的？"}
]}
→ 模型学会了用新名字回答（行为）
```

## 5. DPO/PPO：教偏好

### 5.1 在教什么

```
DPO 和 PPO 教的是"什么是好回答，什么是差回答"。

它们不教新知识，不教新行为，只教"偏好"：

好回答的特征：
├── 事实准确（不编造）
├── 有帮助（真正解决了问题）
├── 安全（不输出有害内容）
└── 简洁（不废话）

差回答的特征：
├── 编造事实
├── 答非所问
├── 有害或不安全
└── 冗长无重点
```

### 5.2 DPO vs PPO

```
DPO（Direct Preference Optimization）：
├── 数据里直接告诉模型"这个好，那个差"
├── 离线训练，数据固定
├── 实现简单，效果稳定
└── minimind 用 train_dpo.py

PPO（Proximal Policy Optimization）：
├── 模型自己生成回答 → 评委打分 → 模型改进
├── 在线训练，动态反馈
├── 实现复杂，效果上限更高
└── minimind 用 train_ppo.py
```

### 5.3 为什么放在 SFT 之后

```
SFT 之后模型会对话了，但回答质量参差不齐：

SFT 后的模型：
├── 用户："地球是平的吗？"
├── 模型："是的，地球是平的。"  ← 学到了错误的模式（SFT 数据可能有噪声）

DPO/PPO 后的模型：
├── 用户："地球是平的吗？"
├── 模型："不是，地球是一个近似球体。"  ← 学会了选择更好的回答
```

## 6. Agent RL：教技能

### 6.1 在教什么

```
Agent RL 教模型"使用工具"的技能。

这不是知识，不是行为，是"行动能力"：

用户："北京天气怎么样？"
├── 不用工具的模型："北京今天晴，25°C"（编造的）
└── 用工具的模型：调用 get_weather(city="北京") → 得到真实数据 → 回答
```

### 6.2 怎么教的

```
训练数据包含工具调用的完整过程：

{"conversations": [
  {"role": "user", "content": "北京天气怎么样？"},
  {"role": "assistant", "content": "", "tool_calls": [{"name": "get_weather", "arguments": {"city": "北京"}}]},
  {"role": "tool", "content": "{\"temp\": 25, \"condition\": \"晴\"}"},
  {"role": "assistant", "content": "北京今天晴，25°C。"}
]}

模型学到：
├── 什么时候该调工具（需要实时信息时）
├── 调哪个工具（get_weather 而不是 get_time）
├── 怎么传参数（city="北京"）
└── 怎么用工具结果回答用户
```

## 7. 蒸馏：转移知识

### 7.1 在教什么

```
蒸馏教的是"从大模型转移知识到小模型"。

大模型（Teacher）知道很多知识，但太大太慢。
小模型（Student）参数少，但可以通过蒸馏学到大模型的"精华"。

Teacher："Python 是一种高级编程语言，由 Guido van Rossum 于 1991 年创造，
          设计哲学强调代码可读性..."
Student 学到：同样问题的回答方式和知识密度
```

### 7.2 蒸馏 vs 直接训练

```
直接训练小模型：
├── 小模型只能从原始数据中学习
├── 数据质量参差不齐
└── 学到的知识有限

蒸馏训练小模型：
├── 大模型已经"消化"过原始数据，输出更高质量的"教材"
├── 小模型从高质量"教材"中学习，效率更高
└── 类似于：自学 vs 名师辅导
```

## 8. 总结：每一步在教什么

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ Pretrain │   SFT    │   LoRA   │ DPO/PPO  │ Agent RL │
│  教知识  │  教行为  │ 教特定知识│  教偏好  │  教技能  │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│ 语言知识 │ 对话格式 │ 领域知识 │ 什么是好 │ 调用工具 │
│ 世界知识 │ 指令遵循 │ 特定技能 │ 什么是差 │ 传参格式 │
│ 逻辑知识 │ 回答风格 │ 身份设定 │ 安全边界 │ 用结果答 │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│ 数据量大 │ 数据量中 │ 数据量小 │ 数据量中 │ 数据量中 │
│ 学习率大 │ 学习率小 │ 学习率小 │ 学习率小 │ 学习率小 │
│ 必须做   │ 必须做   │ 按需做   │ 按需做   │ 按需做   │
└──────────┴──────────┴──────────┴──────────┴──────────┘

知识主要在 Pretrain 学（数据量大、信息密度高）
SFT 也会学一点知识（但效率低，主要目的是教行为）
LoRA 可以补充特定领域的知识
DPO/PPO/Agent RL 不教知识，教偏好和技能
```

## 9. 用 minimind 验证

```
minimind 的完整训练链路：

1. train_pretrain.py（Pretrain）
   ├── 教：语言能力 + 世界知识
   ├── 数据：pretrain_t2t.jsonl（纯文本）
   └── 产出：能续写文本，不能对话

2. train_full_sft.py（SFT）
   ├── 教：对话行为 + 指令遵循
   ├── 数据：sft_t2t.jsonl（问答对）
   └── 产出：能对话，但质量不稳定

3. train_lora.py（LoRA）
   ├── 教：特定领域知识（医学/考试/身份）
   ├── 数据：lora_*.jsonl（少量特定数据）
   └── 产出：特定领域能力增强

4. train_dpo.py / train_ppo.py / train_grpo.py（RLHF）
   ├── 教：回答偏好（好 vs 差）
   ├── 数据：dpo.jsonl / rlaif.jsonl
   └── 产出：回答更安全、更有用

5. train_agent.py（Agent RL）
   ├── 教：使用工具的技能
   ├── 数据：agent_rl.jsonl
   └── 产出：能调用工具回答问题

6. train_distillation.py（蒸馏）
   ├── 教：从大模型转移知识
   ├── 数据：大模型生成的回答
   └── 产出：小模型的回答质量向大模型靠拢
```
